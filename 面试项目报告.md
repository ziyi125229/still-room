# 静室 · Still Room — 项目介绍（STAR 法则 · 面试版）

> 在线体验：https://ziyi125229.github.io/still-room/
> 仓库：github.com/ziyi125229/still-room
> 形态：零依赖单文件 HTML + 图片目录，PWA 可离线、可装成 App。

---

## 一句话

一个**可切换场景的沉浸式自习室**：进入任一场景自动播放对应的自然白噪音，配极简专注工具，帮你"安放此刻的专注"。6 个场景按一天光线排列，纯前端、零依赖、可离线。

---

## STAR

### S · Situation（背景）

专注 / 自习这件事，用户其实要同时凑齐三样东西：**能沉浸的环境画面 + 持续的环境白噪音 + 专注计时工具**。但市面上这三样通常分散在不同 App（壁纸一个、白噪音一个、番茄钟一个），来回切换这个动作本身就在打断专注。

同时，作为 AI 产品经理求职作品集的一块，我想用一个"**小而完整**"的项目证明端到端能力——不只是写 PRD，而是产品定义、设计、工程一个人闭环，并且**真的上线、能被真人点开用**。

### T · Task（目标）

做一个**零依赖、单文件、可离线、可安装**的"沉浸式自习室"：进入即得「场景 + 配套白噪音 + 专注工具」三合一，把"安放专注"做成一个完整闭环。

自加约束（这些约束是后面技术决策的来源）：纯前端、无后端、无第三方库、体积尽量小、断网也能用。

### A · Action（关键行动与决策）

1. **场景情绪动线**：6 个场景按"一天的光线"排列——晨光森林 → 樱花庭院 → 午后咖啡馆 → 海边日落 → 雨夜书房 → 壁炉雪夜。不是随机堆场景，而是给沉浸一条从早到晚的情绪曲线。

2. **双视觉模式（插画 / 实景一键切换）**：每个场景既有**手绘内联 SVG**（矢量、离线、清晰，带雾化景深和蓬松云），也有**本地高清实景照片**。决策理由：SVG 零体积、纯离线、随屏幕缩放都清晰；照片更真实有代入感——两者覆盖不同偏好，让用户自己选。

3. **白噪音全部用 Web Audio 实时合成、零音频文件**（最硬核的决策）：
   - 自己生成 **white / pink / brown 三种底噪 buffer**（pink 用标准滤波系数逼近 1/f、brown 用随机游走积分）。
   - 在底噪上"调"出每种环境音：**雨** = 白噪经 highpass(900Hz)+lowpass(6.5kHz) 叠一层 brown 低频；**海浪** = brown 噪 + 一个 0.12Hz 的 LFO 调制增益，模拟潮汐起伏；**篝火** = brown 底噪 + 随机 bandpass 的"噼啪"爆裂瞬态（指数衰减包络）；**森林** = pink 噪 + 慢 LFO + 随机正弦"鸟鸣"。
   - **为什么这么做**：无音频文件 = 体积极小、纯离线、循环无接缝、可任意叠加混音；用 `setTargetAtTime` 做平滑淡入淡出，节点 `build / teardown` 按需创建销毁省 CPU。
   - 结果是 6 路声音可**自由叠加成自己的混音**，也可"跟随场景"自动播放。

4. **Canvas 粒子叠加**：雨丝 / 落雪 / 光尘 / 海面波光 / 樱花翻飞 / 篝火火星，与场景绑定，让画面"活"起来。

5. **5 个专注工具**（面板可拖动、记忆位置）：番茄钟（今日完成计数、刷新不中断、完成系统通知、与待办联动）/ 正计时（心流）/ 今日要事（标记"正在做"，番茄记在它头上）/ 4-7-8·箱式呼吸（带读秒）/ 声音混音台。

6. **极简抗干扰**：鼠标静止自动隐藏全部界面、进入全屏沉浸；键盘快捷键（空格起停番茄、1–6 切场景、Esc 关面板、F 全屏）。

7. **工程实现**：单个 `index.html`（HTML / CSS / JS 全内联，~80KB）+ `img/`；**PWA**（Web App Manifest + Service Worker 离线缓存 + 可"添加到主屏"）；用 `localStorage` 记忆全部偏好（场景、各音轨音量与开关、待办、今日番茄数）。

### R · Result（结果）

- **已上线、可点开真用、可装成 App**：https://ziyi125229.github.io/still-room/（GitHub Pages 部署）。
- **零依赖 · 零构建 · 单文件**，断网也能用；正文 ~80KB（不含图片）。
- 交付内容：6 场景 × 双视觉模式、6 路可叠加的**程序化**白噪音、5 个专注工具、PWA 离线安装、全键盘操作。
- **体现的能力**：把一个模糊需求（"我想要一个能专注的环境"）收敛成完整可用产品；产品 + 设计 + 工程一个人闭环；并且用"零依赖 / 离线"的约束，**逼出**了有含金量的技术决策（程序化音频合成、内联手绘 SVG）。
- **诚实的局限**：个人作品，目前还没有规模化的真实用户数据。下一步想加一层轻量、隐私友好的统计，看**留存**和**最常用的场景 / 音轨组合**——用数据验证"哪种环境最被需要"，再据此收敛或扩展场景。

## 可能被追问 & 准备好的答案

**Q：白噪音为什么不直接用音频文件？**
A：文件有三个问题——体积大、循环有接缝、不能自由叠加混音。程序化合成全解决了：几乎零体积、纯离线、无限循环无缝、6 路任意叠加，还能按场景动态改参数。代价是要懂一点音频 DSP（噪声染色、滤波、包络、LFO）。

**Q：为什么坚持零依赖单文件？**
A：这是个人作品要"可被任何人一键打开、断网也能用"。约束反而带来好处：没有构建链、部署就是上传一个文件、加载快、可装成 PWA。代价是单文件偏大、协作维护成本高——团队项目我会拆模块 + 上构建。

**Q：手绘 SVG vs 真实照片，为什么两个都做？**
A：SVG 矢量、零体积、纯离线、任意分辨率都清晰，适合"轻、快、统一风格"；真实照片代入感更强。与其替用户选，不如给一键切换，两种偏好都接住。

**Q：这个项目的产品判断体现在哪？**
A：① 识别到"专注"的真痛点是**工具分散打断专注**，所以做"三合一不切换"；② 场景按一天光线排情绪动线，不是堆数量；③ 抗干扰（鼠标静止自动隐藏）是把"沉浸"当一等公民的设计取舍。

**Q：怎么衡量它好不好 / 下一步？**
A：先加隐私友好的轻量统计，看**留存**与**最常用的场景+音轨组合**；用这个信号决定砍哪些场景、加哪些音轨，而不是凭感觉继续堆。
